Introduction

Welcome to the OpenSync documentation! OpenSync is a general-purpose real-time data synchronization library for .NET 8 built with Clean Architecture.

OpenSync — Real-Time Sync Library for .NET

OpenSync provides 5 primitives — Documents, Lists, Maps, Streams, and Channels — that cover every real-time use case from chat and collaborative editing to live dashboards and IoT.

Run it as a standalone microservice or embed it in any ASP.NET Core app in 3 lines of code. Three transport options (WebSocket, SSE, HTTP Long-Polling) give clients flexibility.

The library follows Clean Architecture (Core → Application → Infrastructure → API) with CQRS via MediatR, EF Core persistence (PostgreSQL primary), and optional Redis backplane for multi-instance scaling.

What OpenSync provides that Twilio Sync doesn't
  • Self-hosted, open-source — no vendor lock-in, no per-operation costs
  • Direct database access — own your data completely
  • Multiple transports — SSE and Long-Polling alongside WebSocket
  • Optimistic concurrency with revision-based conflict detection (Twilio uses last-write-wins)
  • Embeddable as a NuGet package in any ASP.NET Core app
  • Pluggable database backends (PostgreSQL, SQLite, SQL Server)
What OpenSync provides that SignalR doesn't
  • 5 high-level sync primitives (Documents, Lists, Maps, Streams, Channels) — SignalR only has RPC Hubs
  • Full REST API for CRUD operations on all primitives
  • Data persistence with PostgreSQL/SQL Server — SignalR is purely in-memory
  • Optimistic concurrency and revision tracking
  • TTL expiry for automatic data cleanup
  • Built-in JWT/API key authentication with scoped permissions
  • Channel presence with metadata and member tracking
  • Rate limiting out of the box

Quick Start

Embed in your ASP.NET Core app
dotnet add package OpenSync.AspNetCore

// Program.cs
using OpenSync.AspNetCore.Extensions;

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddOpenSync(builder.Configuration);

var app = builder.Build();
app.UseWebSockets();
app.UseOpenSync();
app.Run();
Standalone server
git clone https://github.com/Abubakarstu/OpenSync.git
cd OpenSync/src/OpenSync.Api
dotnet run

Swagger UI: https://localhost:5001/swagger

Client SDK

.NET Client
var client = new OpenSyncClient("https://localhost:5001", tokenProvider);

// Document
var doc = await client.Documents.GetOrCreateAsync("settings", new { theme = "dark" });
doc.OnUpdated += (_, args) => Console.WriteLine($"Updated: {args.Data}");
await doc.SetAsync(new { theme = "light", language = "en" });

// Channel
var channel = await client.Channels.GetOrCreateAsync("general");
await channel.JoinAsync(new { name = "Alice" });
channel.OnMessage += (_, args) => Console.WriteLine($"Chat: {args.Data}");
await channel.SendMessageAsync(new { text = "Hello everyone!" });
JavaScript Client
const client = new OpenSyncClient('https://localhost:5001', {
    getToken: async () => {
        const res = await fetch('/api/v1/sync/tokens', {
            method: 'POST',
            body: JSON.stringify({ identity: 'user-123' })
        });
        const data = await res.json();
        return data.data.token;
    }
});
await client.connect();

const doc = client.document('settings');
doc.onUpdated = (data) => console.log('Doc updated:', data);
await doc.set({ theme: 'dark' });

Example — Chat App

var client = new OpenSyncClient("https://localhost:5001", tokenProvider);

var channel = await client.Channels.GetOrCreateAsync("lobby");
await channel.JoinAsync(new { name = "Alice", avatar = "alice.png" });

channel.OnMessage += (_, args) =>
    Console.WriteLine($"[{args.Data["sender"]}] {args.Data["text"]}");

channel.OnMemberJoined += (_, args) =>
    Console.WriteLine($"{args.Metadata["name"]} joined");

await channel.SendMessageAsync(new { sender = "Alice", text = "Hello!" });

Example — Live Dashboard

// Server pushes metrics
var doc = await client.Documents.GetOrCreateAsync("dashboard-metrics");
await doc.SetAsync(new {
    activeUsers = 1423, requestsPerSecond = 89, uptime = "99.97%"
});
// JavaScript client receives updates
const metrics = client.document('dashboard-metrics');
metrics.onUpdated = (data) => {
    document.getElementById('active-users').textContent = data.activeUsers;
    document.getElementById('rps').textContent = data.requestsPerSecond;
};

Architecture Layers

┌─────────────────────────────────────────────┐
│              API / Presentation             │
│  REST Endpoints · WebSocket Hub · SSE · LP  │
├─────────────────────────────────────────────┤
│               Application Layer             │
│  Commands · Queries · Sync Orchestration    │
├─────────────────────────────────────────────┤
│                  Domain Layer               │
│  SyncObject · Revision · TTL · Permissions  │
├─────────────────────────────────────────────┤
│               Infrastructure Layer          │
│  EF Core · PostgreSQL · Identity · Cache    │
└─────────────────────────────────────────────┘

Core: Domain entities & value objects (SyncObject, Revision, Subscription). Application: SyncOrchestrator, token generation, pagination. Infrastructure: EF Core (PostgreSQL), ASP.NET Identity, in-memory cache. API: Controllers, WebSocket (SignalR), SSE, Long-Polling middleware.

Event Flow

Client A ──PUT──► Server ──► DB ──► SyncOrchestrator ──► Notify ──► Client B
                    │                                           │
                    ├── SSE  ◄──────────────────────────────────┘
                    ├── WebSocket (SignalR)
                    └── Long-Polling

Client A writes → Server persists & updates revision → SyncOrchestrator resolves subscribers → Event dispatched via real-time transport to Client B. Optimistic concurrency: revision mismatch → 409 Conflict.

Deployment

Docker
# Build
docker build -t opensync-server .

# Run with PostgreSQL
docker run -d -p 5000:8080 \
  -e ConnectionStrings__DefaultConnection="Host=pg;Database=opensync;User ID=postgres;Password=..." \
  opensync-server
docker-compose
services:
  api:
    image: opensync-server
    ports: ["5000:8080"]
    environment:
      - ConnectionStrings__DefaultConnection=Host=pg;Database=opensync;...
      - JwtSettings__Secret=your-jwt-secret
    depends_on: [pg]
  pg:
    image: postgres:16
    environment:
      POSTGRES_DB: opensync
      POSTGRES_PASSWORD: your-password
    volumes: ["pgdata:/var/lib/postgresql/data"]
volumes: { pgdata: }
Cloud (Azure)

Deploy the API as an Azure App Service, use Azure Database for PostgreSQL Flexible Server, and enable Azure SignalR Service for scaled-out WebSocket connections.
